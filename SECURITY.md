# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue first.

Send details to the maintainers with:

- Description of the issue
- Affected files/components
- Reproduction steps or proof of concept
- Suggested mitigation (if available)

We will acknowledge receipt and work on triage and remediation.

## Supported Versions

This repository is under active development. Security fixes are applied to the current default branch and recent release line when applicable.

## Sensitive Data Guidance

- Never commit API keys, credentials, or service account files.
- Use environment variables and GitHub Secrets.
- Rotate any secret immediately if exposed.
