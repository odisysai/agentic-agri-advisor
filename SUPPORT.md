# Support

## Getting Help

Use the following channels for help:

- Open a GitHub Discussion for design, usage, setup, or architecture questions.
- Open a GitHub Issue for reproducible bugs, documentation gaps, or feature requests.
- For security concerns, follow the private reporting process in `SECURITY.md`.

Please do not report security vulnerabilities through public issues.

## What to Include

To help maintainers triage quickly, include:

- Environment details:
  - Operating system
  - Browser and version
  - Python version
  - Node.js version
  - Device type, if mobile or tablet
- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs, screenshots, or screen recordings
- Whether the issue occurs in:
  - Local development
  - Hosted demo
  - Mobile browser
  - Installed PWA
  - Guest mode
  - Google sign-in mode

## Agriculture and Safety Questions

Krishi Sampark is a capstone/demo project and should not be treated as a replacement for professional agricultural advice.

For questions involving:

- Fertilizer quantities
- Pesticide or chemical usage
- Crop disease treatment
- Financial decisions
- Safety-critical farm operations

please consult a qualified agriculture expert, extension officer, or local authority.

Do not submit private farmer data, sensitive personal information, or confidential field information in public issues.

## Demo Limitations

The hosted demo may include sample data, mock integrations, or limited functionality. Some features may be marked as roadmap or prototype.

When reporting a demo issue, please mention:

- The URL used
- Whether you used Guest Mode or Google sign-in
- Selected language
- Device type and screen size
- Approximate time of the issue

## Response Expectations

This is an actively developed capstone project. Response times may vary based on maintainer availability, issue severity, and project phase.

Priority is generally given to:

1. Security issues
2. Safety-related agriculture guidance issues
3. Bugs affecting app access or authentication
4. Mobile usability issues
5. Documentation and enhancement requests

## Community Guidelines

Please follow the project `CODE_OF_CONDUCT.md` when asking for help or participating in discussions.
