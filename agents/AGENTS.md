## `agents/agent.py` — ADK Entry Point (DO NOT CONFUSE WITH A REAL AGENT)

`agents/agent.py` is a **thin re-export shim** required by `agents-cli` and the ADK runner for package-level discovery. It contains only:

```python
from agents.coordinator.agent import coordinator_agent as root_agent
```

**Do NOT edit this file when adding a new specialist agent.** It should always re-export `coordinator_agent` as `root_agent` and nothing else. The real coordinator logic lives in `agents/coordinator/agent.py`.

## Coding Rules

### 1. Agent Definition

```python
from google.adk.agents import Agent
from google.adk.models import Gemini

my_agent = Agent(
    name="my_agent",                          # snake_case, unique across all agents
    model=Gemini(model="gemini-2.5-flash"),   # DO NOT change unless explicitly asked
    instruction="...",
    tools=[],
)
```

### 2. LANGUAGE RULE (mandatory in every specialist agent)

Every specialist agent instruction must contain this exact block, updated for all supported language codes:

```
LANGUAGE RULE: You MUST respond in the language specified by the coordinator.
If instructed 'Respond in hi', reply in Hindi.
If instructed 'Respond in mr', reply in Marathi.
If instructed 'Respond in te', reply in Telugu.
If instructed 'Respond in sw', reply in Swahili.
If instructed 'Respond in zu', reply in Zulu.
```

When a new language is added, this block must be updated in ALL 5 specialist agent files.

### 3. Tool Imports

Import the tool instance, not the module:
```python
# CORRECT
from google.adk.tools.load_web_page import load_web_page

# WRONG
import google.adk.tools.load_web_page
```

### 4. Persona Rule

No agent may expose its own name, role label, or the multi-agent architecture to the farmer in any response. All responses go through the coordinator's unified persona (Krishi Bisesagya).

### 5. Model Fix Note

If you see `Model 404` or location errors:
- Fix `GOOGLE_CLOUD_LOCATION` env var — try `global` instead of `us-east1`.
- Do NOT change the model string.

---

## Prohibited Actions

- Do not modify `before_agent_callback` or `after_agent_callback` on the coordinator.
- Do not expose agent routing logic or internal sub-agent names in responses.
- Do not hardcode API keys or credentials in any agent file.
- Do not change the coordinator's Farmer Mode JSON format without updating the frontend parser.
